# 我要填的清单（耳朵日报 / 本地阅读站）

## 1) 我关心的平台 KEY（必填）

位置：`my/RSS.md`

- [ ] 我想要的 KEY（1 行 1 个，支持 “知乎 / V2EX / 雪球 / 少数派 / 阮一峰…”）
- [ ] 我不想要的 KEY（删除即可）

## 2) 运行参数偏好（可选）

默认直接运行即可（会自动读取 `RSS源.md` + `my/RSS.md`，并生成 `NewsReport/` + `site/`）。

- [ ] GitHub Trending 窗口：`daily / weekly / monthly`（默认 `daily`）
- [ ] 每个源抓多少条候选：`--per-feed-limit`（默认 auto-mode=30）
- [ ] 总抓取时间预算秒：`--time-budget`（默认 auto-mode=240）
- [ ] 每个平台最多保留：`--per-platform-limit`（默认在 key 模式下=10）
- [ ] 质量门槛：`--min-score`（默认 2.6）
- [ ] 需要强制 IPv4 吗：`--prefer-ipv4`（默认 auto-mode=开）

## 3) AI 摘要（可选）

- [ ] 我是否要开 AI：填 `OPENAI_API_KEY`
- [ ] 我用的模型：`OPENAI_MODEL`（默认 `gpt-4o-mini`）

## 4) 站点外观（可选）

文件：`site/index.html`

- [ ] 站点标题（当前：耳朵日报）
- [ ] 副标题（当前：本地阅读站…）
- [ ] 是否改字体（当前：Fraunces + IBM Plex Sans，来自 Google Fonts）

## 5) 我怎么打开网站（必选其一）

- [ ] 直接双击：`site/index.html`（部分浏览器对 `file://` 可能有限制）
- [ ] 本地起服务（推荐）：`python3 -m http.server 5173 --directory site`

## 6) 失败源处理（可选）

如果你看到 report 里某些源长期失败：

- [ ] 我是否接受删除/替换这些源（在 `my/RSS.md` 里移除对应 KEY，或换成可用的镜像源）
- [ ] 是否允许添加“备用抓取”（比如从同平台的其他 RSS/镜像补位）
